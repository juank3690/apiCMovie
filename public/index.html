Documentación de API DMDB

La API DMDB es una plataforma que permite gestionar y consultar información sobre películas, series de TV, personas y contenido en tendencia. Está desarrollada con Node.js y Express y proporciona diversos endpoints para interactuar con estos datos.

Base URL

http://localhost:3000/api

Endpoints Principales

Movies

GET {{baseUrl}}/movie

Descripción: Obtener todas las películas.

Respuesta de Ejemplo:

[
    {
        "id": 1,
        "title": "Movie Title",
        "release_date": "2024-01-01"
    }
]

GET {{baseUrl}}/movie/{{movieId}}

Descripción: Obtener detalles de una película específica.

Respuesta de Ejemplo:

{
    "id": 1,
    "title": "Movie Title",
    "release_date": "2024-01-01"
}

TV Shows

GET {{baseUrl}}/tv

Descripción: Obtener todas las series de TV.

Respuesta de Ejemplo:

[
    {
        "id": 1,
        "name": "TV Show Title",
        "first_air_date": "2024-01-01"
    }
]

GET {{baseUrl}}/tv/{{tvId}}

Descripción: Obtener detalles de una serie de TV específica.

Respuesta de Ejemplo:

{
    "id": 1,
    "name": "TV Show Title",
    "first_air_date": "2024-01-01"
}

People

GET {{baseUrl}}/people

Descripción: Obtener todas las personas.

Respuesta de Ejemplo:

[
    {
        "id": 1,
        "name": "Person Name",
        "known_for": "Acting"
    }
]

GET {{baseUrl}}/people/{{peopleId}}

Descripción: Obtener detalles de una persona específica.

Respuesta de Ejemplo:

{
    "id": 1,
    "name": "Person Name",
    "known_for": "Acting"
}

Trending

GET {{baseUrl}}/trending

Descripción: Obtener contenido en tendencia.

Respuesta de Ejemplo:

[
    {
        "id": 1,
        "title": "Trending Title",
        "type": "movie"
    }
]

Configuración de Postman

Para configurar Postman para usar esta API:

  1. Crea una nueva colección en Postman llamada API DMDB.
  2. Crea un nuevo entorno en Postman llamado Local y añade las siguientes variables:
    • baseUrl: http://localhost:3000/api
    • movieId, tvId, peopleId: Deja estos valores vacíos inicialmente y actualízalos después de hacer las solicitudes.

Instrucciones para las Pruebas

  1. Selecciona el método adecuado (GET, POST, etc.) en Postman.
  2. Introduce la URL correspondiente usando las variables definidas ({{baseUrl}}/movie, etc.).
  3. Añade cualquier parámetro necesario en las secciones correspondientes (Params, Headers, Body).
  4. Haz clic en "Send" y verifica la respuesta.

Ejemplo de Solicitud en Postman

GET {{baseUrl}}/movie

Respuesta esperada:

[
    {
        "id": 1,
        "title": "Movie Title",
        "release_date": "2024-01-01"
    }
]

Conclusión

Esta API proporciona varios endpoints para gestionar y consultar información sobre películas, series de TV, personas y contenido en tendencia. Utiliza las variables y métodos adecuados en Postman para realizar pruebas exhaustivas y asegurar el correcto funcionamiento de la API.
